# Codex 呈现对齐清单（Codex Parity Backlog）

> 目标：把 DualCode 中与 Codex 工具重叠的功能（对话回合、思考、命令执行、文件变更、审批、
> 中断、回合元信息）的**样式、交互与行为对齐到 Codex 的真实呈现**。
> 供 Codex 按序执行，Claude 负责逐项 review，用户以并排对比验收。
> 执行约定沿用 `docs/REMEDIATION_BACKLOG.md` 全部规则：一条目一 commit（编号 + 中文说明）、
> 每条附测试、禁止顺手重构、全量验证、每个条目完成即停等 review 或按条目说明推进。
>
> 基线说明：`93ae36d`/`7f25fe2`/`b508dae` 已完成文档式消息布局（去卡片、跨头像占位、
> 与输入框对齐），本清单在该基线上继续。

## 事实来源与验收方式（重要）

「完全一样」的事实来源是 **Codex CLI 当前版本的真实呈现**，不是本文档作者的记忆。
因此设立 C0 校准条目：**执行者（Codex）最了解自己的界面，先产出对照基线，用户确认后才实施**。
每个条目的最终验收都是用户把 DualCode 与 Codex CLI 并排跑同类任务对比。
GUI 与终端媒介不同的地方（鼠标悬停、可点击展开等），保留 GUI 惯例，但**信息结构、
文案语气、默认展示/折叠策略**必须与 Codex 一致。

## 保护条款（不得违反）

1. **审批安全语义不得放宽**：对齐的是审批的展示与选项文案，不是审批范围；
   高风险操作逐次审批的规则（PROJECT_STATUS 安全不变量）保持不变。
2. 近期 U 系列成果中与 Codex 风格冲突的（如 U4 思考胶囊、活动卡折叠语义），
   **以 Codex 风格为准**，但必须在 C0 清单中明确列出"将被替换的现有行为"，
   逐项替换并改造对应测试，不允许静默删除测试。
3. Claude（VPS SSH）消息的呈现不在本清单范围——本清单只对齐 Codex 回合。

## 执行节奏

1. C0 完成后**停下**，等用户确认基线；
2. C1 是结构性最大的一条，完成后**停下**等 Claude review + 用户目验；
3. C2-C7 每条独立 commit，连续推进，全部完成后统一 review；
4. 所有条目完成后，用户做一轮完整的并排对比验收，结论记入本文档。

## 明确不做

- Claude 消息呈现的改动（另立条目再议）。
- Codex TUI 独有且无 GUI 对应物的行为（如终端滚动缓冲、shell 转义渲染）。
- /slash 命令面板、会话恢复选择器等 Codex 独立功能的移植（属新功能，非重叠功能对齐）。
