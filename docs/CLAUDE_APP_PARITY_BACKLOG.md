# Claude 桌面应用风格对齐清单（Claude App Parity Backlog）

> 目标：把 DualCode 与 Claude 桌面应用重叠的界面（对话回合、思考、工具/命令展示、
> 消息操作、输入区、审批、配色气质）对齐到 Claude 桌面应用的样式与交互。
> 规格由 Claude 直接给出（本文档即事实来源），Codex 按序执行，Claude 逐项 review，
> 用户以与 Claude 桌面应用的并排观感对比做最终验收。
> 执行约定沿用 `docs/REMEDIATION_BACKLOG.md` 全部规则（一条目一 commit、附测试、
> 禁止顺手重构、全量验证）。本文档取代已删除的 CODEX_PARITY_BACKLOG.md（参照对象勘误）。
>
> 基线：文档式消息布局（`93ae36d`/`7f25fe2`/`b508dae`）与 U1-U6 成果已就位，
> 其中多项（代码块、思考胶囊、流式策略）本就以 Claude App 为参照，本清单做到"完全对齐"。

## 保护条款

1. **审批安全语义不得放宽**：对齐展示与文案，不改审批范围与后端语义。
2. 配色对齐的是"暖中性 + 单一强调色"的气质，**不使用 Anthropic 的品牌标识/名称/Logo**。
3. 与 Claude App 无对应物的功能（右侧检查器、VPS 面板等）不在范围内，保持现状。

### A1 对话列布局与用户消息

**规格（Claude App 行为）**：单列居中，内容列宽约 720-768px；用户消息为**右对齐圆角容器**
（radius 约 12-16px，背景比页面高一级明度，最大宽度约列宽 75%），无头像无名字头；
助手回复为**通栏文档式纯文本**（无容器、无边框、无头像列），与用户气泡同列对齐；
回合之间以留白分隔（无分隔线）。用户消息 hover 时右下浮现小号"编辑"入口。

- [ ] 校准现有布局到上述规格：列宽、气泡最大宽度/圆角/明度、回合间距统一为 token；
  当前 `--conversation-content-width: 900px` 收窄到 ~760px。
- [ ] 用户消息 hover 编辑：点击进入就地编辑态（textarea 替换气泡内容，保存=重发该轮，
  取消恢复），替代现有"编辑后重新发送"按钮跳转输入框的方式；"重试本轮"保留在同一 hover 入口。
- **验收**：组件测试覆盖就地编辑的保存/取消；并排观感对比由用户确认。

### A2 思考块（Extended Thinking 形态）

**规格**：思考进行中——一行浅色标签「正在思考…」带微光/脉动效果，其下**默认展开**流式
思考文本（小一号、次级颜色、非斜体、左侧无边框线，仅靠缩进与颜色区分）；思考结束——
自动收起为一行「已思考 X 秒」+ 右侧 chevron 的行内条目（非胶囊按钮形态，是通栏可点击行），
点击展开完整思考文本；展开态与流式态视觉一致。

- [ ] 现有 thinking-block/thought-pill 调整为上述形态：去掉斜体与左边框线，改缩进+次级色；
  「已思考 X 秒」由 999px 胶囊改为通栏行内可点击条目；展开/收起 150ms 过渡。
- [ ] 思考标签的脉动效果替换现有光标符号（▍保留给正文流式，思考不用）。
- **验收**：组件测试覆盖流式态/收起态/展开回看三态文案与结构；用户并排对比。

### A3 工具与命令执行块

**规格**：每次工具调用/命令执行是一个**独立圆角容器行**（radius ~10px，背景高一级明度，
无边框）：左侧小图标 + 工具名/动作短语（如「运行命令」「修改文件」），右侧状态
（运行中 spinner / 完成对勾 / 失败叉），整行可点击展开；展开区显示等宽输入（命令全文）
与输出（默认尾部若干行，可再展开全部）。多个调用纵向堆叠，间距 6-8px，**不包外层折叠卡**。

- [ ] 将现有 ActivityCard 的「已处理 N 秒 + 折叠步骤时间线」重构为上述独立行堆叠形态：
  回合内工具行直接内联在消息流中、默认可见（Claude App 不隐藏工具行），
  仅每行的输入/输出细节需点击展开。
- [ ] 命令输出接入现有 terminal 通道数据（该命令对应的输出片段；若协议无法按命令切分，
  在「执行中新发现」记录并降级为「查看运行日志」跳转）。
- [ ] 失败行显示简短错误摘要（红色次级文本），替代现有独立 activity-error 区。
- **验收**：组件测试覆盖运行中/成功/失败/展开四态与堆叠顺序；用户并排对比。

### A4 消息操作（重新引入，位置修正）

**规格**：助手消息的操作条位于**消息末尾下方**（不悬浮遮挡正文——此前顶部悬浮方案已因

复制、重试两个图标按钮，带 tooltip；点击复制后图标短暂变对勾。

- [ ] 在助手消息末尾添加底部操作行（复制 Markdown 原文 / 重试本轮），复用

- [ ] 该方案与 `062696d` 的移除决定不冲突（问题是"顶部悬浮遮挡"，本方案底部占位不遮挡）；

### A5 输入区（Composer）

**规格**：大圆角容器（radius ~16-20px），内部上下两行：上行 textarea（自动增高），
下行左侧「+」附件按钮与模式选择（小号、无边框、hover 显底色），右侧**圆形发送按钮**
（向上箭头图标，主强调色背景；运行中变为方块停止图标）；无常驻快捷键提示文案
（提示放 placeholder 或 tooltip）；附件缩略图行在 textarea 上方。

- [ ] 调整现有 Composer 到上述结构与形态：圆形主按钮、快捷键提示移入 tooltip、
  模式选择改为无边框小号样式；现有拖放高亮、IME 守卫、草稿、禁用逻辑全部保留。
- **验收**：现有 Composer 测试全绿 + 新增圆形按钮状态测试；用户并排对比。

### A6 审批卡（权限请求形态）

**规格**：Claude App 的工具权限请求是**内联卡片**：标题「允许 X？」，正文区等宽块展示
完整命令/操作内容，按钮行右对齐——主按钮「允许」（强调色实底）、次按钮「本次任务均允许」
（仅编辑类操作出现）、文字按钮「拒绝」；卡片圆角与 A3 工具行一致，出现时 150ms 淡入。

- [ ] 现有 ApprovalCard 按上述重排：标题句式、等宽内容块、按钮层级（主/次/文字）与顺序；
  选项语义与后端映射不变（保护条款 1）。
- **验收**：审批组件测试更新；后端审批测试全绿；用户并排对比。

### A7 配色与气质（暖中性化）

**规格**：Claude App 深色模式的气质是**暖中性**：背景近 #262624/#1f1e1b 一族（偏暖的深灰，
非蓝黑）、正文米白 #f5f4ef 一族、次级文本暖灰、强调色为陶土橙一族（约 #d97757），
用于主按钮/发送键/活跃态；边框极少，层级靠明度。当前 DualCode 是冷蓝灰系。

- [ ] 在 U5 主题层重定义 surface/文本/强调 token 为暖中性系（列出具体色值供 review），
  强调色替换现有蓝色主按钮；代码块/终端保持中性深灰不跟随暖化。
- [ ] Codex/Claude 消息的身份区分色（现左边框已移除）改为仅名字标签着色，色相收敛到
  与暖色系协调的两个低饱和度色。
- [ ] **单独一个 commit**，完成后暂停等用户截图确认（同 U5 惯例），不满意整体 revert。
- **验收**：字号/对比度策略测试通过（对比度不得低于现状）；用户截图确认。

### A8 空状态与系统提示

**规格**：空对话居中问候语（大号、暖色调）而非功能说明卡；系统事件（测试结果等）为
低调的行内灰色短句，不用绿色边框盒。

- [ ] 新任务空状态改为居中问候 +（可选）示例提示语；system 消息样式降噪为行内灰字。
- **验收**：组件测试更新；用户确认。

## 执行节奏

1. A7 配色是全局观感的地基但风险最大，**放最后**做（其余条目在现配色下先对齐结构）；
2. A1 → A2 → A3 是结构主线，每条完成即停等 Claude review；A3 完成后用户做一次中期目验；
3. A4 需先贴形态说明经用户点头（见条目内说明）；
4. A5/A6/A8 连续推进，统一 review；最后 A7 单独提交 + 用户截图验收。

## 明确不做

- Anthropic 品牌元素（Logo、名称、字体 Styrene/Tiempos）的使用。
- Claude App 独有功能的移植（Artifacts、Projects、模型选择器语义）。
- 右侧检查器、VPS 面板、恢复中心等无对应物界面的改动。
- Light 主题（维持既有范围外决定）。
